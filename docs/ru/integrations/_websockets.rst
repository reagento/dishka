В большинстве случаев мы работаем с отдельными событиями, например, HTTP-запросами. В этом случае у нас есть только две области (scope): ``APP`` и ``REQUEST``.
Вебсокеты устроены иначе: для одного приложения существует множество соединений (по одному на каждого клиента), и каждое соединение передаёт множество сообщений.
Для поддержки этой модели мы используем дополнительную область: ``SESSION``:

    ``APP`` |rarr| ``SESSION`` |rarr| ``REQUEST``
